Preference files are used by Java applications to store information about settings and preferences defined by the user for a particular application. JavaNews uses a single file called javanews.prefs.xml

Entries within the preference file are organised within a tree and all of the entries for JavaNews can be found grouped together under a particular branch.

The entries for JavaNews are all stored in plain-text so the XML file can be opened like any other. JavaNews uses many different entries but of particular importance are:

Entry ID Purpose
javanews.lnf The look and feel to be used for the application.
javanews.logLevel The level of events to be included in log files created by JavaNews.
javanews.dir.Log The directory where log files should be placed.
javanews.dsn.name The name of the database to connect to. If you are using ODBC then this should be the DSN name.
javanews.comPort.numberToMonitor The number of COM ports being monitored by the application. This is used to retrieve the rest of the COM port settings.
javanews.x.name The name of the COM port to connect to. The x in the ID is replaced by a number which is zero-indexed.
javanews.x.baud The baud rate of the COM port that JavaNews should connect to.
javanews.x.parity The parity type of the COM port.
javanews.x.data The data bit length of the COM port.
javanews.x.stop The stop bit length of the COM port.
