JavaNews has been designed and built in the Java language. It makes use of features implemented in Java Standard Edition (SE) v1.6.0 and higher. A copy of Java has been included for ease in the program directory, this is the 1.6.0 update 11 release. Newer releases can be found at Java.com.

The installation of the Java SE runtime edition is straight-forward and you should just follow the on-screen instructions and accept the default settings where needed (you may want to change default installation path). Once complete it is recommended to restart the PC to ensure that the system has been updated correctly.

Add-on libraries

In order for JavaNews to function correctly you must install two add-ons to the standard Java libraries. These add-ons are RxTx and JFreeChart which have been written by 3rd parties. More information about both of these add-ons can be found both in the Bibliography and Licensing information pages that can be found in the menu on the right.

If you use the automated installation you will not need to manually install these files.

During the installation of Java you should noted which installation directory is used for the new version (for Windows platforms this should be along the lines of C:\Program Files\Java\jrexxxx, where xxxx is the version being installed).

Within the Java installation directory you should find the \lib\ext folder and the \bin folder. These are the two directories used to install add-on libraries. You can find the files that require copying into these locations in the Libraries folder on the JavaNews installation CD. The library files section below lists the files, their purpose and which folder they should be copied to.

Installing newer versions of Java

If you need to update your installed version of Java to a newer version, you must ensure that all of the add-on files are copied to their respective folders to ensure that JavaNews works with the new version of Java.

Installing a new version of Java will not automatically move add-on libraries to the new version.

Library files

These are the files required for the add-ons. The column "Must Be Located In" shows which folder within your Java installation the folder the file should be placed in. So, if your version of Java is located in C:\Program Files\Java\jre1.6.0_11\ then the file rxtxSerial.dll should be placed in the folder C:\Program Files\Java\jre1.6.0_11\bin\:

File Name Part Of Purpose Must Be Located In
rxtxParallel.dll RxTx Allows access to parallel ports in Microsoft Windows .\bin
rxtxSerial.dll RxTx Allows access to serial (COM) ports in Microsoft Windows .\bin
RXTXcomm.jar RxTx The libraries for RxTx to work with Java .\lib\ext
jcommon-1.0.12.jar JFreeChart Provides text and utility classes as a foundation for the charting classes .\lib\ext
jfreechart-1.0.9.jar JFreeChart Provides the graphing and charting classes required for JFreeChart .\lib\ext

If you are using a prior version of Java

You can install a new version of Java into a different folder from the earlier installation. If problems arise during execution then you should uninstall both versions and reinstall the newer, however this is unlikely.

If you are using Microsoft JVM or a Virtual Machine from another manufacturer

You must install the Sun Java SE Runtime. The Virtual Machine used by Sun contains the whole Java specification and other manufacturers can prove incompatible with native code.
