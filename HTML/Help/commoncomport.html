How to identify the problem

If the application receives data from the COM port then the indicator in the bottom right-hand corner of the screen will flash. This indicates that the application has received a string of characters terminated by the newline character (ASCII code 13). On receipt of a new message the application will check it and attempt to process it. If there are any problems with the received message then it is rejected. Multiple rejections where the status light is flashing but without any new messages being added to the COM port window is indicative of the COM port being incorrectly configured.

The problem is apparent when the raw data window is opened. The entries contained within it are garbled and not displayed correctly if the COM port is not set properly.

What causes the problem

Changes to the configuration either at the transmitting end or receiving end of the transmission will lead to these types of problems.

How to fix it

Changing the settings under Configuration > COM Port Configuration will remedy the problem.
