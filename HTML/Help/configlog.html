The logging system in JavaNews will output details based on errors and events encountered within the application code. The different entries into the logs are given varying levels of severity based on how important they are. These are listed below in descending importance (the more important ones are at the top and less so are at the bottom):

  1. Severe
  2. Warning
  3. Info
  4. Config
  5. Fine
  6. Finer
  7. Finest

JavaNews uses all of the above levels when logging events and this can give a detailed log of what is happening within the application which is useful for debugging when something goes wrong.

Logging levels

Within the configuration menu the level of all the logging levels can be set by selecting the desired level from the list.

By selecting an entry from the menu you will set the loggers to record any events at that level and those that are higher in the list. This would mean that if you set the level to Config any Severe, Warning, Info and Config level events would be stored in the log files.

The current level is show with a black diamond (using the Nimbus Look and Feel) or similar next to the level name.

When the changes take effect

As the loggers are a core part of the application it is not possible to change them "on the fly". The changes are stored in the preferences file and will take effect the next time the application is started.

The application must be restarted before a change of the logging level takes effect.
