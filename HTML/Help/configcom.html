The COM ports used by JavaNews can be set and configured using this screen. It is accessible through the Configuration menu and is listed as "COM Port Config". By using this screen you are able to:

  • Specify which COM ports should be monitored for incoming PSE messages
  • Configure the settings to be used on the port
  • Enable or disable the test port (shown as COM99)

You can specify which ports to monitor by clicking on the check box on the left-hand side to toggle whether the port will be monitored. The "in use?" column shows whether the port is being used by another application. The port must be free before it can be used by JavaNews.

The values for baud rate, data bits, parity type and stop bits can be chosen from drop-down boxes. This ensures that invalid values cannot be entered into the application.

Once you have made any changes, click on Save to store the settings in the preferences file for future reference. Any ports that you have chosen to monitor that weren't previously open will have a corresponding window open in the main application window.

Using the test port

The test port within the application can be enabled by checking the box next to the entry named COM99. The other settings for this port are ignored.

The test port reads the contents of the OpenNewsSample.txt file that is contained within the root installation directory of JavaNews. The port reads all of the lines of the file and then feeds them to the application as if it were a genuine COM port. This can be used for testing whether a PSEs output will be accepted by the application following configuration changes, updates or other changes.
