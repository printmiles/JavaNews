How to identify the problem

Messages received during one session are not displayed on a query in another which should result in the messages being shown.

What causes the problem

There are multiple problems that can cause this to happen, the common point of these is the database. The application can be prevented from saving data either because the database won't allow the application to connect or the database doens't exist or has been moved.

How to fix it

As the problems relate to the database then checking the configuration of the ODBC data source name will verify and hopefully resolve the cause of the problem. The information here can be used to check the ODBC configuration. You may need to enter a user name and password to access the database (these aren't required for the database supplied with the application).
