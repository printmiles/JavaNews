In the window for each COM port being monitored you will find that information is summarised within three tables. These tables are split across the screen in the center and right-hand sides:

Received messages (center)

This contains a summary of information about the messages received on a COM port. This summary includes (from left to right):

  • The severity of the message
  • The name of the PSE that generated the message
  • The identifier of the link on the named PSE that generated the message
  • The message contents that have been received
  • The date, according to the PSE, when the message was sent
  • The unique identifier of the message within the application

Recognised links (top-right)

This contains the name of the PSE and link ID for each of the recognised links along with its current status displayed as a colour:

  • Grey - Link state unknown: A message has not been received that explicitly mentions the links state. PSE nodes themselves will stay in this state.
  • Red - Link down: A message has been received that specifically mentions this link as being down (LINK DOWN).
  • Green - Link up: A LINK UP message has been received explicitly stating that the link is up and ready.

Discovered links (bottom-right)

This displays the same information as for recognised links. However a link which has sent a message to this COM port which isn't known in the recognised links table will be entered here. You can "acknowledge" discovered links which moves them into the recognised table and allows you to run reports on them. For more information on how to do this, click here.

Detailed information

You are able to view details of the records displayed within JavaNews. You can do this by double-clicking on an entry that is showing a recognised link, discovered link or a received message. The details you can see depends on the type of entry you selected, whether messages or links.

Recognised & discovered links

If you double-click on a link (listed in the two table on the right-hand side of the JavaNews window) which is either recognised (top) or discovered (bottom) then you will see more information about the link displayed in a window like this:

Within this screen you can see the following information about the link:

  • The COM port the link is configured on
  • The PSE name the link is attached to
  • The link ID within the PSE
  • The details of the link
  • A description of the link
  • The current status of the link (link up, link down or unknown)
  • When the status of the link last changed

You are also able to add and edit links but this is covered in the editing records page which can be found here.

Received messages

The details of received messages show not only the information received from the PSE but also related information. This information can include comments about the message, the definition of the message received and when the message was actually received.

To see the status of the link which generated this message you can click on the "Show Link" button. This will display the link details screen shown previously.

Should you wish to add a comment to the message you can click on the "Add Comment" button. Doing this will display an input box where you can enter a brief comment. The comment can be up-to 255 characters including spaces, more than this will lead to the comment being truncated. Once you have finished entering your comment, click on OK and the message will be displayed in the table.
